
# 程序改造优化说明
	1. 解决跨网段情况下丢包过多的问题，sorket实现方式的更改为python自带的包（pysyslogclient）的方式发送syslog日志，稳定一些
	2. 修改一个调测时候的速率bug（在计算速率时候的sleep的位置移动）
	3. 修改为可配置 和 可外部传参启动方式，优势:  在Windows / linux下也可以后台启动适合长时间的稳定性测试（已提供一键启停脚本）
	4. 改进为写日志方式，优势：后台长时间运行时候可以实时记录速率和日志总量
	5. 新增日志配置文件 log2file.py，可根据自己需要修改，目前日志 50M 切换
	6. 在main.py文件 pysyslogclient.SyslogClientRFC5424('192.168.4.165', 514, proto="UDP")  提供了主机， 端口 和 协议内容，目前UDP测试已完成，理论上TCP方式也是可用的，本次优化对TCP未做测试，可根据需要自己修改。





---------------------------------------------------------------Windows---------------------------------------------------------------

1. 读配置文件的启停脚本：
	
	run_conf.vbs 、start_conf.bat、stop.bat
	
	说明： 使用过程中修改好配置文件datamodel.py之后， 双击 run_conf.vbs 脚本 调测速率，发送日志，启动之后，在Windows的任务管理器的详细信息里面可以看到python.exe的进程，停止 双击 stop.bat 即可
	
2. 参数带入方式的启停脚本：

	run_data.vbs 、start_data.bat、stop.bat
	
	说明： 使用过程中修改start_data.bat 入参之后， 双击 run_data.vbs 脚本 调测速率，发送日志，启动之后，在Windows的任务管理器的详细信息里面可以看到python.exe的进程，停止 双击 stop.bat 即可
